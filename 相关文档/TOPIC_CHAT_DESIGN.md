# 随手一记 · 话题对话系统设计方案

> 将当前的"发布-列表"模式重构为"话题-对话"模式
> 日期：2026-06-27

---

## 一、设计理念

### 现状 vs 目标

| 维度 | 现状（发布流） | 目标（话题对话） |
|------|--------------|----------------|
| **内容组织** | 按时间排列的单条笔记 | 按话题聚合的多轮对话 |
| **交互方式** | 发布→列表浏览 | 创建话题→好友回复→对话流 |
| **社交关系** | 单向展示 | 双向互动 |
| **数据模型** | `note` 单层 | `topic` → `reply` 双层 |
| **视觉风格** | 信息流卡片 | 聊天气泡 |

### 核心场景

```
用户 A 创建话题："周末一起去晨跑吗？"
  → 好友 B 回复："几点？在哪集合？"
  → 用户 A 回复："6点半，南体育场"
  → 好友 C 回复："我来！"
```

```javascript
  nickname: '好友B',
  content: '6点半太早了，7点行吗？',

```

### API 接口

```
GET    /api/topics            — 话题列表（支持分页/筛选）
POST   /api/topics            — 创建话题
GET    /api/topics/:id        — 话题详情（含回复列表）
DELETE /api/topics/:id        — 删除话题

GET    /api/topics/:id/replies     — 回复列表（分页）
POST   /api/topics/:id/replies     — 回复话题
DELETE /api/replies/:id            — 删除回复
```

## 五、交互细节

| 操作 | 动效 | 反馈 |
|------|------|------|
| 创建话题 | 底部 Sheet 上滑弹出，200ms ease | 键盘自动弹出 |
| 发布话题 | 按钮 loading → 成功 Toast → 列表插入 | 列表项交错入场 |
| 进入话题 | 右滑进入详情页，标题过渡 | 回复列表从底部出现 |
| 发送回复 | 气泡立即插入（乐观更新），灰底 → 正常 | 输入框清空 |
| 新消息 | 话题卡片显示"new"标记 | 回复数+1 |
| 删除 | 滑动确认 → 左滑出现删除按钮 | 确认弹窗 |

## 六、实施步骤

### 阶段一：后端 API（~1h）

| 任务 | 说明 |
|------|------|
| 创建 `topics` 数据表 | id, user_id, content, tags, visibility, files, reply_count, created_at |
| 创建 `replies` 数据表 | id, topic_id, user_id, content, files, created_at |
| 实现话题 CRUD API | 列表/创建/详情/删除 |
| 实现回复 API | 列表/创建/删除 |
| 话题列表支持筛选 | 全部 / 我的话题 / 好友话题 |

### 阶段二：前端页面（~1.5h）

| 任务 | 说明 | 工时 |
|------|------|------|
| 创建 `TopicList.vue` | 话题列表页（替换 Notes.vue） | 25min |
| 创建 `TopicDetail.vue` | 话题详情对话页（新页面） | 30min |
| 创建 `CreateTopicSheet.vue` | 创建话题底部弹窗 | 15min |
| 创建 `ReplyBubble.vue` | 聊天气泡组件 | 10min |
| 创建 `topicStore.js` | Pinia store 管理话题状态 | 10min |
| 创建 `replyApi.js` | 回复 API 封装 | 5min |

### 阶段三：过渡与打磨（~30min）

| 任务 | 说明 |
|------|------|
| 气泡入场动画 | 每气泡 100ms 交错延迟 |
| 乐观更新 | 发送后立即显示，失败回滚 |
| 未读标记 | 话题卡片显示新回复数 |
| 空状态 | "还没有话题，创建一个吧" |

## 十、总结

| 维度 | 当前（笔记流） | 目标（话题对话） |
|------|--------------|----------------|
| 交互模型 | 发布 → 浏览 | 创建话题 → 好友回复对话 |
| 社交深度 | 单向展示 | 双向互动 |
| 页面数 | 1（列表） | 2（列表 + 详情） |
| 数据复杂性 | 单层 note | 双层 topic + reply |
| 视觉独特性 | 一般 | ⭐⭐⭐⭐⭐ |
| 实施难度 | — | 中（后端+前端约 3h） |
| 用户粘性 | 低 | ⭐⭐⭐⭐⭐ 高 |

> 此方案将"随手一记"从**个人日记**转化为**社交话题广场**，
> 每个话题成为一个独立的对话空间，好友可以在其中自由交流。
