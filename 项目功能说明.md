# KEEPro - Keep运动截图生成器 项目功能说明

## 项目概述

**KEEPro** 是一个基于浏览器的 **Keep（跑步/运动）打卡截图生成工具**。用户可以通过修改页面上的各项参数（如日期、时间、公里数、配速、温度、湿度、天气图标等），生成一张仿 Keep APP 风格的运动打卡截图，并支持下载为 PNG 图片。同时支持手动或自动绘制运动轨迹路径，并可批量生成多张不同日期/参数的截图。

- **版本**: v3.1
- **项目地址**: [https://gitee.com/joytion/keep/](https://gitee.com/joytion/keep/)
- **技术栈**: 纯前端（HTML + CSS + JavaScript），使用 html2canvas 库生成截图

## 核心功能模块详解

### 1. 用户信息设置（√ 标记项 — 可持久化存储）

| 参数 | 说明 | DOM元素 |
|------|------|---------|
| 用户名 | 显示在截图上的用户昵称 | `#inpt_username` |
| 头像 | 用户头像图片，支持自定义上传 | `#portrait` / `#inpt_portrait` |
| 标题 | Keep标题，如"户外跑步" | `#inpt_keep_title` |

### 2. 运动数据设置（#rand/#live 标记项 — 实时随机）

| 参数 | 说明 |
|------|------|
| 公里数 | 跑步里程，支持设置随机范围（min~max），也可手动输入固定值 |
| 配速 | 每公里用时（如 5'30"），支持随机范围 |
| 日期时间 | 年/月/日 时/分，可手动设置或由程序自动获取当前时间 |
| 消耗热量 | 根据公里数自动计算 |
| 运动时长 | 根据公里数和配速自动计算 |

### 3. 天气信息设置

- **天气图标**: 支持三种预设天气 — 晴☀️(`weather1.png`)、多云⛅(`weather2.png`)、阴天☁️(`weather3.png`)
- **温度**: 手动输入或通过高德地图API自动获取
- **湿度**: 手动输入或通过高德地图API自动获取
- **高德地图API Key**: 输入API Key后可自动获取实时的IP定位城市天气数据

### 4. 背景与轨迹设置

#### 背景选择
- **预设场地**: 南体育场、军工操场、北体育场（三个场地，各有不同的背景图）
- **自定义背景**: 支持上传自定义背景图片
- **预设轨迹**: 南体育场提供5种预设的运动轨迹路径图片

#### 轨迹绘制系统（核心特色功能）
这是项目最复杂的功能模块：

- **手动绘制**: 点击"点击绘制"按钮，打开Canvas画布，用户可用鼠标/触摸自由绘制轨迹
- **随机生成**: 调用后端API `https://tool.joytion.cn/generate-track` 获取随机运动轨迹JSON数据
- **批量生成**: 使用本地 `drawingActions.json` 文件（1100+坐标点），通过 `fetch` 拦截机制在本地产生多样化轨迹，无需依赖远程API
- **路径变色**: 可启用在轨迹中随机插入色块变化效果（模拟Keep APP的轨迹颜色渐变）
  - 变色概率(`bs_prob`): 每段轨迹变色的概率
  - 变色步长范围(`bs_range_min`~`bs_range_max`): 变色持续多少步
- **起点/终点标记**: 自动在轨迹起点放置"开始"图标，终点放置"结束"图标
- **自动绘画**: 页面加载时可自动从API获取随机轨迹并绘制（由"自动"复选框控制）

#### 轨迹JSON数据格式（drawingActions.json）
```json
[
    {"action": "down", "x": 45.08, "y": 223.86},   // 笔按下
    {"action": "move", "x": 56.34, "y": 222.69},   // 笔移动
    ...
    {"action": "up",   "x": 145.60, "y": 209.99}   // 笔抬起
]
```

### 5. 图片下载功能

提供 **4种下载方案**：

| 方案 | 方法 | 说明 |
|------|------|------|
| Download1 | `html2canvas` + `canvas.toBlob()` | 主要方案，保存为PNG图片 |
| Download2 | `html2canvas` + Canvas转BlobURL | 在新标签页中查看图片 |
| Download3 | `html2canvas` + `canvas.toDataURL()` | 备用方案1 |
| Download4 | `html2canvas` + Blob URL下载 | 备用方案2（手动重命名） |

- 支持自定义图片保存宽度（`#inpt_savePic_width`）

### 6. 数据持久化（IndexedDB）

使用浏览器 IndexedDB 数据库存储以下数据：

- **user_info**: 用户名、标题、公里数范围、配速范围、背景图选择、天气设置、轨迹变色设置、图片宽度、自动绘画开关、高德API Key
- **user_portrait**: 用户头像图片（Base64）
- **user_bgimg**: 自定义背景图片（Base64）

页面加载时自动从 IndexedDB 恢复所有持久化数据，点击"永存带√信息"按钮手动保存当前设置。

### 7. 批量生成截图功能（NEW）

通过 `batch_generate.js` 实现，位于页面下半部分的面板：

| 参数 | 说明 |
|------|------|
| 生成数量 | 1~100张 |
| 日期范围 | 起始日期 ~ 结束日期（均匀分布） |
| 时间范围 | 每天的时间范围（随机分布） |
| 天气模式 | 固定天气 / 随机天气（晴/多云/阴天） |
| 温度范围 | 随机温度范围（℃） |
| 湿度范围 | 随机湿度范围（%） |
| 轨迹模式 | 随机轨迹（每张不同）/ 保持当前轨迹 |

**技术亮点**: 随机轨迹模式下，通过拦截 `window.fetch` 对 `tool.joytion.cn/generate-track` 的请求，使用本地缓存的 `drawingActions.json` 数据施加随机坐标偏移（`perturbTrackData`），在本地生成多样化轨迹，无需依赖后端API，避免网络错误。

## 技术实现详解

### 截图生成流程

```
1. 用户修改各项参数
2. 调用 render() 函数 → 更新DOM中所有显示元素
3. 调用 html2canvas → 将截图预览区域渲染为Canvas
4. Canvas → toDataURL/toBlob → 下载为PNG文件
```

### 轨迹绘制流程

```
1. inpt_drawbtn_onClick() → 打开轨迹绘制弹窗
2. 用户手动绘制 或 Json2Draw(API/本地数据) → 自动绘制
3. Canvas上绘制轨迹 + 起点/终点标记
4. drawpic_yesbtn_onClick() → 将Canvas内容合成到背景图 → 关闭弹窗
5. 后续截图生成时会包含已绘制的轨迹
```

### 动态数据计算

- **运动时长** = 公里数 × 配速（自动换算为 HH:MM:SS 格式）
- **配速显示** = 小数分钟转换为 `分'秒"` 格式
- **热量消耗** = 公里数 × 约60千卡（粗略估算）
- **自动绘画** = 页面加载时从 `https://tool.joytion.cn/generate-track` 获取随机轨迹
